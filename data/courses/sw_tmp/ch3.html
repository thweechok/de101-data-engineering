1. ทำความรู้จักกับ The Box Model (กล่องแห่งความลับ)

ถ้ามีคอนเซปต์เดียวใน CSS ที่คุณต้องเข้าใจให้ถ่องแท้ สิ่งนั้นคือ The Box Model กฎเหล็กของหน้าเว็บคือ: "ทุกๆ Element ใน HTML จะถูกมองว่าเป็นกล่องสี่เหลี่ยม 1 ใบเสมอ" ไม่ว่ารูปร่างหน้าตาที่คุณเห็นจะเป็นวงกลม (ผ่านการใช้ border-radius) หรือเป็นข้อความธรรมดา ลึกๆ แล้วมันคือกล่องสี่เหลี่ยม

กล่องสี่เหลี่ยม 1 ใบนี้ ประกอบไปด้วย 4 เลเยอร์ (จากในสุดออกมานอกสุด) ดังนี้:

  1. Content (เนื้อหา): แกนกลางของกล่อง คือพื้นที่ที่ใช้แสดงข้อความ รูปภาพ หรือวิดีโอ (กำหนดขนาดด้วย width และ height)
  2. Padding (พื้นที่ว่างด้านใน): พื้นที่ที่กั้นระหว่าง Content กับ Border ทำหน้าที่ "ดันเนื้อหาเข้าข้างใน" ช่วยให้เนื้อหาไม่ชิดขอบจนเกินไป และรับสีพื้นหลังเดียวกับ Content
  3. Border (เส้นขอบ): เส้นที่ล้อมรอบ Padding และ Content เป็นเส้นแบ่งขอบเขตของกล่อง คุณสามารถปรับขนาด สไตล์ และสีของเส้นได้
  4. Margin (พื้นที่ว่างด้านนอก): พื้นที่ที่อยู่นอกสุด ทำหน้าที่ "ดันกล่องอื่นๆ ออกไป" เพื่อสร้างระยะห่างระหว่าง Element (Margin เป็นพื้นที่โปร่งใส ไม่มีสีพื้นหลัง)

2. ปัญหาโลกแตก และพระเอกที่ชื่อว่า box-sizing: border-box

ตามค่าเริ่มต้นของเบราว์เซอร์ (Standard Box Model) เมื่อคุณกำหนดความกว้าง (width) ให้กล่อง มันจะนับแค่ส่วน Content เท่านั้น นั่นแปลว่า ถ้าคุณกำหนด width: 100px; แล้วไปเพิ่ม padding: 20px; และ border: 5px; ขนาดความกว้างที่แท้จริงของกล่องบนหน้าจอจะกลายเป็น 100 + (20*2) + (5*2) = 150px!

สิ่งนี้ทำให้การคำนวณ Layout น่าปวดหัวมาก เพราะกล่องจะขยายเกินกว่าที่เราคาดการณ์และล้นหน้าจอ (Overflow) เสมอ เพื่อแก้ปัญหานี้ นักพัฒนาเว็บทั่วโลกจะใช้คำสั่งวิเศษเพื่อเปลี่ยนพฤติกรรมการคำนวณนี้ โดยใช้ box-sizing: border-box; ซึ่งจะบังคับให้ width ที่เรากำหนด รวม Padding และ Border เข้าไปแล้ว (ถ้ากำหนด 100px กล่องก็จะกว้าง 100px เป๊ะ ส่วนพื้นที่ Content ข้างในจะถูกบีบให้เล็กลงแทน)

/* CSS Reset ยอดฮิตที่ทุกโปรเจกต์ต้องมี */
* {
  box-sizing: border-box;
  margin: 0;
  padding: 0;
}

3. Display Property (รูปแบบการแสดงผลของกล่อง)

คุณสมบัติ display ใช้บอกเบราว์เซอร์ว่ากล่องนี้ควรมีพฤติกรรมการวางตัวแบบไหนในหน้าเว็บ ค่าที่สำคัญที่สุดมี 4 แบบ:

  • block: กล่องจอมตระกละ มันจะกินพื้นที่ความกว้าง (width) 100% ของบรรทัดนั้น และผลัก Element ถัดไปให้ขึ้นบรรทัดใหม่เสมอ (เช่น แท็ก <div>, <p>, <h1>) กล่องประเภทนี้สามารถกำหนด width และ height ได้อย่างอิสระ
  • inline: กล่องผู้เจียมตัว กินพื้นที่ความกว้างเท่าที่จำเป็นตามเนื้อหาข้างใน ไม่บังคับขึ้นบรรทัดใหม่ ข้อเสียคือ ไม่สามารถกำหนด width, height, margin-top หรือ margin-bottom ได้ (เช่น แท็ก <span>, <a>, <strong>)
  • inline-block: ร่างผสมที่สมบูรณ์แบบ มันสามารถเรียงต่อกันในบรรทัดเดียวได้เหมือน inline แต่ในขณะเดียวกันก็สามารถกำหนด width, height, margin, padding ได้เหมือน block (เหมาะกับการทำปุ่ม หรือแท็ก <img>)
  • none: กล่องล่องหน ซ่อน Element นี้ไปเลยราวกับว่าไม่มีอยู่จริง (ถูกลบออกจาก Layout ทั้งหมด ต่างจาก visibility: hidden; ที่ซ่อนแค่ตาเห็น แต่ยังจองพื้นที่ Layout เดิมอยู่)
💡 Box Model Debugging Tip เวลาที่หน้าเว็บคุณพัง เลย์เอาต์เลื่อน หรือมีแถบ Scrollbar แนวนอนโผล่มาแบบงงๆ ให้ลองแทรกโค้ด * { border: 1px solid red !important; } ลงไปใน CSS เพื่อตีเส้นขอบสีแดงให้ทุกกล่องในหน้าเว็บ คุณจะเห็นทันทีว่าตัวการคือกล่องใบไหนที่ล้นหรือใหญ่เกินความจำเป็น!
