1. ทำความรู้จักกับ HTML5

HTML5 เป็นเวอร์ชันล่าสุดและเป็นมาตรฐานสากลของ HyperText Markup Language ที่ถูกพัฒนาขึ้นเพื่อรองรับฟีเจอร์ของเว็บยุคใหม่ (Modern Web) รวมถึงการรองรับมัลติมีเดีย เช่น วิดีโอและเสียง โดยไม่ต้องพึ่งพาปลั๊กอินอย่าง Flash โครงสร้างพื้นฐานของการเขียน HTML จะประกอบไปด้วย 3 คำศัพท์หลักที่คุณต้องจำให้ขึ้นใจ คือ Tags, Elements, และ Attributes

2. Tags, Elements และ Attributes คืออะไร?

  • Tag (แท็ก): คือคำสั่งที่ใช้เพื่อบอกเบราว์เซอร์ว่าข้อความหรือเนื้อหานั้นๆ คืออะไร แท็กจะถูกครอบด้วยวงเล็บมุม (Angle brackets) เช่น <p> มักจะมาเป็นคู่เสมอ คือ Tag เปิด (Opening tag) และ Tag ปิด (Closing tag) ซึ่งแท็กปิดจะมีเครื่องหมาย Slash (/) เพิ่มเข้ามา เช่น </p>
  • Element (อิลิเมนต์): คือองค์ประกอบโดยรวมทั้งหมด ตั้งแต่ Tag เปิด, เนื้อหา (Content) ที่อยู่ตรงกลาง, ไปจนถึง Tag ปิด เช่น <p>สวัสดีชาวโลก</p> ทั้งหมดนี้เรียกว่า 1 Paragraph Element
  • Attribute (แอตทริบิวต์): คือคุณสมบัติหรือข้อมูลเพิ่มเติม (Metadata) ที่เรากำหนดให้กับ Element นั้นๆ แอตทริบิวต์จะถูกเขียนไว้ใน Tag เปิดเสมอ และมักมาในรูปแบบ name="value" เช่น <a href="https://example.com">คลิกที่นี่</a> ในที่นี้ href คือ Attribute ที่บอกว่าลิงก์นี้จะชี้ไปที่ไหน

3. โครงสร้างพื้นฐานของไฟล์ HTML (HTML Boilerplate)

ทุกไฟล์ HTML ต้องมีโครงสร้างพื้นฐานที่เหมือนกัน เพื่อให้เบราว์เซอร์ทำงานได้อย่างถูกต้อง ดังนี้:

<!DOCTYPE html>
<html lang="th">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>เว็บไซต์แรกของฉัน</title>
</head>
<body>
  <!-- เนื้อหาของเว็บไซต์จะอยู่ที่นี่ -->
</body>
</html>
  • <!DOCTYPE html>: ประกาศว่าไฟล์นี้เป็น HTML5
  • <html>: Root Element ครอบทุกอย่างในเอกสาร
  • <head>: ส่วนหัวที่เก็บข้อมูล Metadata, ลิงก์ไฟล์ CSS, ตั้งค่า SEO (ผู้ใช้จะไม่เห็นเนื้อหาในส่วนนี้บนหน้าเว็บ)
  • <body>: ส่วนลำตัวที่เก็บเนื้อหาทั้งหมดที่ต้องการให้ผู้ใช้มองเห็น เช่น ข้อความ รูปภาพ วิดีโอ

4. Semantic HTML: การใช้แท็กให้มีความหมาย

ในอดีต นักพัฒนาเว็บมักจะใช้ <div> (Division) หรือ <span> ในการครอบเนื้อหาทุกส่วน ซึ่งทำให้โค้ดอ่านยาก และคอมพิวเตอร์ไม่เข้าใจว่าส่วนไหนสำคัญ HTML5 จึงริเริ่มแนวคิด Semantic HTML ซึ่งคือการใช้แท็กที่มีความหมายตรงกับเนื้อหา

ข้อดีของ Semantic HTML คือ:

  1. SEO (Search Engine Optimization) ดีขึ้น: Google หรือ Search Engine อื่นๆ สามารถให้คะแนนและเข้าใจโครงสร้างเว็บได้ดีขึ้น
  2. Accessibility (การเข้าถึง): โปรแกรม Screen Reader สำหรับผู้พิการทางสายตาสามารถอ่านและนำทาง (Navigate) ผ่านเว็บไซต์ได้ง่ายขึ้น

Semantic Tags ที่สำคัญ:

  • <header>: ส่วนหัวของหน้าเพจหรือบทความ มักมีโลโก้และแถบเมนูนำทาง
  • <nav>: ส่วนที่บรรจุลิงก์สำหรับการนำทางหลัก (Navigation menu)
  • <main>: เนื้อหาหลักของหน้าเพจ (ควรมีแค่ 1 อันต่อ 1 หน้า)
  • <article>: เนื้อหาที่สมบูรณ์ในตัวเอง สามารถแยกไปอ่านเดี่ยวๆ ได้ เช่น บทความบล็อก, โพสต์ข่าว
  • <section>: การแบ่งกลุ่มเนื้อหาเป็นส่วนๆ โดยเนื้อหาในกลุ่มนั้นต้องมีความเกี่ยวข้องกัน
  • <footer>: ส่วนท้ายของหน้าเพจ มักเป็นข้อมูลลิขสิทธิ์, ช่องทางการติดต่อ, ลิงก์ย่อยๆ

5. Forms and Inputs (การรับข้อมูลจากผู้ใช้)

หนึ่งในฟีเจอร์ที่สำคัญที่สุดของเว็บคือ การสร้างแบบฟอร์ม เพื่อรับข้อมูลจากผู้ใช้ (User Input) เช่น ฟอร์มล็อกอิน, สมัครสมาชิก, หรือค้นหาสินค้า เราใช้แท็ก <form> เป็นตัวครอบ input ต่างๆ

<form action="/submit-data" method="POST">
  <label for="username">ชื่อผู้ใช้:</label>
  <input type="text" id="username" name="user_name" required placeholder="กรอกชื่อของคุณ">
  
  <label for="userEmail">อีเมล:</label>
  <input type="email" id="userEmail" name="user_email" required>
  
  <button type="submit">ส่งข้อมูล</button>
</form>

HTML5 มาพร้อมกับ Input Types ใหม่ๆ เช่น type="email", type="date", type="number" ซึ่งเบราว์เซอร์จะทำการ Validation (ตรวจสอบความถูกต้อง) ให้เบื้องต้นโดยอัตโนมัติ เช่น หากใส่ type="email" แล้วผู้ใช้พิมพ์ข้อความที่ไม่มีเครื่องหมาย @ เบราว์เซอร์ก็จะแจ้งเตือนก่อนส่งฟอร์ม

💡 Best Practice สำหรับ Forms ควรใช้แท็ก <label> คู่กับ <input> เสมอ โดยกำหนดให้ค่า Attribute for ของ label ตรงกับค่า id ของ input สิ่งนี้จะช่วยให้ผู้ใช้สามารถ "คลิกที่ข้อความ" เพื่อเป็นการโฟกัสที่ช่อง input ได้ทันที (ลองนึกถึงช่อง Checkbox เล็กๆ การคลิกที่ข้อความข้างๆ จะช่วยให้ผู้ใช้งานผ่านมือถือคลิกได้ง่ายขึ้นมาก)
