1. ทำไมเราถึงต้องมีตัวแปรใน CSS?

ลองจินตนาการว่าคุณต้องดูแลโปรเจกต์เว็บไซต์ขนาดใหญ่ที่มีหน้าเว็บมากกว่า 50 หน้า และแบรนด์ของคุณใช้ "สีน้ำเงินสด (#007bff)" เป็นสีธีมหลัก (Primary Color) คุณจะต้องเขียนโค้ดสีนี้ซ้ำๆ ในปุ่ม, ลิงก์, แถบเมนู หรือเส้นขอบ นับร้อยนับพันครั้ง วันหนึ่งทีม Marketing ตัดสินใจรีแบรนด์และสั่งให้เปลี่ยนธีมเป็น "สีม่วง" คุณคงต้องนั่งค้นหาและแทนที่ (Find & Replace) จนตาแฉะ และเสี่ยงที่จะมีข้อผิดพลาดสูงมาก

นี่คือจุดที่ CSS Variables (หรือชื่อเป็นทางการคือ CSS Custom Properties) เข้ามาเป็นฮีโร่ขี่ม้าขาว มันทำให้คุณสามารถเก็บค่าใดๆ ก็ได้ (สี, ขนาดฟอนต์, ระยะห่าง) ไว้ใน "ตัวแปร" ที่ส่วนกลางที่เดียว แล้วเรียกใช้ตัวแปรนั้นทั่วทั้งโปรเจกต์ หากวันหนึ่งต้องการเปลี่ยนแปลง ก็แค่ไปแก้ค่าตัวแปรนั้นที่จุดเดียว ทุกที่ที่เรียกใช้ก็จะเปลี่ยนตามทันที!

2. วิธีการประกาศและเรียกใช้งาน (Syntax)

การสร้างตัวแปรใน CSS จะต้องตั้งชื่อโดยใช้เครื่องหมายขีดกลางสองตัวนำหน้า -- (Double Dash) เสมอ เช่น --primary-color และเรานิยมประกาศตัวแปรส่วนกลางไว้ใน Pseudo-class ที่ชื่อว่า :root ซึ่งเป็นตัวแทนระดับสูงสุดของหน้าเว็บ (เทียบเท่าแท็ก html) เพื่อให้ตัวแปรเหล่านี้สามารถถูกมองเห็นและเรียกใช้ได้จากทุกที่ (Global Scope)

/* 1. ประกาศตัวแปร (เปรียบเสมือนพจนานุกรมส่วนตัว) */
:root {
  --primary-color: #007bff;
  --secondary-color: #6c757d;
  --main-padding: 15px;
  --base-radius: 8px;
}

/* 2. เรียกใช้งานด้วยฟังก์ชัน var() */
.button-primary {
  background-color: var(--primary-color);
  padding: var(--main-padding);
  border-radius: var(--base-radius);
  color: white;
}

.button-primary:hover {
  /* สามารถกำหนด Fallback (ค่าสำรอง) ได้เป็นพารามิเตอร์ที่สอง เผื่อกรณีที่พิมพ์ชื่อตัวแปรผิดหรือหาไม่เจอ */
  background-color: var(--primary-hover, blue); 
}

3. Scope (ขอบเขตการเข้าถึง) ของตัวแปร

ตัวแปร CSS มีเรื่องของขอบเขต (Scope) คล้ายกับภาษาโปรแกรมมิ่งทั่วไป หากคุณประกาศตัวแปรไว้ใน :root มันจะมองเห็นได้ทั้งหน้าเว็บ (Global) แต่ถ้าคุณประกาศไว้ใน Selector เฉพาะเจาะจง มันก็จะใช้ได้แค่ใน Element นั้นและลูกหลานของมันเท่านั้น (Local Scope)

.card-special {
  --card-bg: #ffeedd; /* ตัวแปรนี้จะใช้ได้เฉพาะข้างใน .card-special เท่านั้น */
  background-color: var(--card-bg);
}
/* หากเรียก var(--card-bg) ที่อื่น จะไม่ได้ค่านี้ไป */

4. พลังที่แท้จริง: การทำ Theming และ Dark Mode

ข้อได้เปรียบสูงสุดของ Native CSS Variables (เมื่อเทียบกับตัวแปรของ Preprocessor อย่าง SASS/LESS) คือ มันมีชีวิตและสามารถเปลี่ยนแปลงค่าได้แบบ Real-time บนเบราว์เซอร์ โดยไม่ต้องคอมไพล์โค้ดใหม่ ทำให้การทำ Theme สีหลายๆ แบบ หรือทำ Dark Mode กลายเป็นเรื่องที่ง่ายดายมาก

เราสามารถใช้ Media Query prefers-color-scheme ซึ่งจะคอยเช็คว่าระบบปฏิบัติการ (Windows, iOS, macOS) ของผู้ใช้ตั้งค่าเครื่องเป็นโหมดมืด (Dark Mode) หรือไม่ จากนั้นเราก็แค่ "สลับค่า" ของตัวแปร

/* Light Mode (ค่าเริ่มต้นของระบบ) */
:root {
  --bg-color: #ffffff;
  --text-color: #333333;
}

/* สลับร่างเป็น Dark Mode ถ้าผู้ใช้เปิดโหมดมืดในเครื่อง */
@media (prefers-color-scheme: dark) {
  :root {
    --bg-color: #1a1a1a;
    --text-color: #f1f1f1;
  }
}

/* นำไปใช้งานเพียงครั้งเดียว ระบบจะสลับสีพื้นหลังและตัวอักษรให้เองอัตโนมัติ! */
body {
  background-color: var(--bg-color);
  color: var(--text-color);
}
💡 การใช้งานร่วมกับ JavaScript CSS Variables เปิดประตูบานใหญ่ให้ CSS พูดคุยกับ JavaScript ได้อย่างราบรื่น คุณสามารถใช้ JS ดึงค่าตัวแปรมาคำนวณ หรือแก้ไขค่าได้อย่างง่ายดายผ่าน document.documentElement.style.setProperty('--primary-color', 'red') สิ่งนี้ทำให้เราสามารถสร้างลูกเล่นล้ำๆ อย่างเช่น การให้ผู้ใช้กด Color Picker เลือกสีธีมเว็บเอง แล้วเว็บทั้งหน้าเปลี่ยนสีทันที!
