1. ข้ามขีดจำกัดด้วย CSS Grid (เลย์เอาต์ 2 มิติ)

ในขณะที่ Flexbox เกิดมาเพื่อจัดการเลย์เอาต์แบบ 1 มิติ (จัดการได้ทีละแถว หรือ ทีละคอลัมน์) CSS Grid Layout ถูกพัฒนาขึ้นมาเพื่อความยิ่งใหญ่กว่านั้น มันคือระบบจัดการเลย์เอาต์แบบ 2 มิติ (2-Dimensional) ที่ทรงพลังที่สุดใน CSS ซึ่งสามารถจัดการจัดวางคอนเทนต์ทั้ง แถว (Rows) และ คอลัมน์ (Columns) ได้พร้อมๆ กันในคราวเดียว

เพื่อให้เห็นภาพ ให้จินตนาการว่า CSS Grid คือการตีตารางตารางหมากรุกลงบนหน้าเว็บ แล้วคุณเป็นคนสั่งการว่า "กล่องนี้ให้ไปวางที่ช่องมุมซ้ายบน", "กล่องนี้ให้ขยายร่างกินพื้นที่ 3 ช่องตรงกลาง" มันจึงเหมาะอย่างยิ่งสำหรับการวางโครงร่าง (Macro Layout) ภาพรวมของเว็บไซต์ทั้งหน้า เช่น การทำเลย์เอาต์ที่มี Header, Sidebar ด้านซ้าย, Content หลักตรงกลาง, และ Footer ด้านล่าง

2. การสร้าง Grid Container และการตีตาราง

ขั้นตอนแรกเหมือนกับ Flexbox คือเราต้องเปลี่ยนกล่องแม่ให้เป็นตารางก่อนด้วย display: grid; จากนั้นเราต้องนิยามกฎว่าตารางของเราจะมีกี่คอลัมน์ กี่แถว และขนาดเท่าไหร่

.grid-container {
  display: grid;
  grid-template-columns: 200px 1fr 1fr;
  grid-template-rows: 100px 300px;
  gap: 20px; /* ระยะห่างระหว่างช่องตาราง */
}
  • grid-template-columns: กำหนดขนาดและจำนวนของคอลัมน์ (จากตัวอย่างคือสร้าง 3 คอลัมน์)
  • grid-template-rows: กำหนดขนาดและจำนวนของแถว (จากตัวอย่างคือสร้าง 2 แถว)
  • gap: ช่องไฟ หรือร่องระหว่างกล่อง (Gutter) ช่วยให้คุณไม่ต้องมานั่งตั้ง margin ทีละตัวเพื่อสร้างระยะห่างอีกต่อไป

3. ทำความรู้จักกับหน่วยวิเศษ "fr" (Fractional Unit)

Grid นำเสนอหน่วยวัดใหม่ที่เกิดมาเพื่อมันโดยเฉพาะ นั่นคือ fr (Fraction) ซึ่งหมายถึง "สัดส่วนของพื้นที่ว่างที่เหลืออยู่" จากตัวอย่าง 200px 1fr 1fr หมายความว่า: คอลัมน์แรกจะกว้าง 200px แบบตายตัว ส่วนพื้นที่ที่เหลือทั้งหมดในหน้าจอ จะถูกแบ่งออกเป็น 2 ส่วนเท่าๆ กัน (1fr และ 1fr) เพื่อมอบให้กับคอลัมน์ที่ 2 และ 3 สิ่งนี้ทำให้การทำ Responsive Layout ง่ายขึ้นมหาศาล โดยที่เราไม่ต้องคำนวณสัดส่วนเป็นเปอร์เซ็นต์ (%) ให้ปวดหัวเมื่อมีการปรับเปลี่ยนหน้าจอ

4. การวางตำแหน่ง Grid Items (ควบคุมลูกๆ ให้อยู่ในร่องในรอย)

เมื่อเราตีตารางที่กล่องแม่เสร็จแล้ว เราสามารถบอกลูกแต่ละตัวได้ว่าให้ไปสิงสถิตอยู่ช่องไหน หรือกินพื้นที่กว้างกี่ช่อง โดยอ้างอิงจาก Grid Lines (เส้นตาราง)

ระวัง: การนับในระบบ Grid จะนับจาก "เส้นขอบ" ไม่ใช่ "ช่อง" โดยเส้นแรกสุดฝั่งซ้ายคือเส้นที่ 1 เสมอ

.header {
  /* ลากตั้งแต่เส้นคอลัมน์ที่ 1 ไปจนถึงเส้นที่ 4 (ผลคือการกินพื้นที่ 3 คอลัมน์) */
  grid-column: 1 / 4; 
}
.sidebar {
  /* วางอยู่ระหว่างเส้นคอลัมน์ 1 ถึง 2 และเส้นแถวที่ 2 ถึง 3 */
  grid-column: 1 / 2;
  grid-row: 2 / 3;
}

5. ความมหัศจรรย์ของการวาดเลย์เอาต์ด้วย Grid-Template-Areas

ฟีเจอร์ที่เจ๋งที่สุดของ Grid คือ Grid Areas ซึ่งช่วยให้เราสามารถ "วาดภาพ" เลย์เอาต์ด้วยตัวอักษรได้เลยใน CSS Code ซึ่งเป็นวิธีที่อ่านแล้วเห็นภาพตามได้ชัดเจนที่สุด (Visual Layout)

.container {
  display: grid;
  grid-template-areas:
    "header header header"
    "sidebar content content"
    "footer footer footer";
}

.header-item { grid-area: header; }
.sidebar-item { grid-area: sidebar; }
.content-item { grid-area: content; }
.footer-item { grid-area: footer; }
💡 Flexbox vs Grid: สรุปแล้วจะเลือกใช้อะไรดี? กฎทองของการจัดเลย์เอาต์ยุคใหม่คือ: ใช้ Grid สำหรับ Layout ภาพรวมสเกลใหญ่ (Macro Layout) เช่น การจัดวางส่วนประกอบหลักของหน้าเพจที่ต้องควบคุมทั้งแกน X และ Y อย่างเคร่งครัด และ ใช้ Flexbox สำหรับส่วนประกอบย่อยๆ (Micro/UI Components) เช่น การเรียงปุ่มในแถบเมนู (Nav Bar), การจัดกลุ่มไอคอน หรือการจัดหน้าตาของการ์ดสินค้า (Product Card)
ไม่จำเป็นต้องเลือกฝั่ง โปรเจกต์ที่ดีที่สุดคือโปรเจกต์ที่ผสมผสานนำข้อดีของทั้ง Flexbox และ Grid มาใช้ร่วมกันอย่างลงตัว!
