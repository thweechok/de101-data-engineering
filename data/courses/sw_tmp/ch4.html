1. Normal Flow (กระแสหลักของหน้าเว็บ)

ก่อนจะพูดถึงการจัดตำแหน่ง (Positioning) ขั้นแอดวานซ์ เราต้องเข้าใจก่อนว่า ถ้าเราไม่เขียน CSS ใดๆ เลย เบราว์เซอร์จะนำเสนอ Element ต่างๆ อย่างไร? พฤติกรรมดั้งเดิมนี้เรียกว่า Normal Flow (หรือ Document Flow)

ใน Normal Flow กล่องประเภท block จะถูกวางเรียงซ้อนกันจากบนลงล่าง (Top to Bottom) เหมือนบล็อกไม้ ในขณะที่กล่องประเภท inline จะเรียงต่อกันจากซ้ายไปขวา (Left to Right) เหมือนการเขียนตัวหนังสือ การใช้ CSS Positioning คือวิธีการเอา Element "หลุด" ออกมาจาก Normal Flow เพื่อให้เราสามารถวางมันตรงไหนก็ได้ตามใจชอบบนหน้าจอ

2. ชนิดของ Position (Types of Positioning)

Property position มีค่าที่ใช้งานบ่อย 5 รูปแบบ เมื่อเราเปลี่ยนค่า position เป็นค่าอื่นที่ไม่ใช่ static เราจะได้รับพลังวิเศษในการใช้ property ควบคุมพิกัด 4 ตัว ได้แก่ top, bottom, left, และ right เพื่อดันกล่องไปในทิศทางต่างๆ

  1. static (ค่าเริ่มต้น): กล่องอยู่ใน Normal Flow ปกติ ไหลไปตามยถากรรม ไม่สามารถใช้ top, bottom, left, right ได้เลย
  2. relative (อ้างอิงตำแหน่งเดิมตัวเอง): กล่องยังคงอยู่ใน Normal Flow (ยังจองพื้นที่เดิมของมันไว้) แต่มันสามารถถูกดันให้ขยับออกจาก "ตำแหน่งเดิมที่มันควรจะอยู่" ได้ เช่น top: 20px; left: 10px; จะดันกล่องลงมา 20px และเลื่อนไปขวา 10px จากจุดเดิม
  3. absolute (อ้างอิงบรรพบุรุษผู้ทรงพลัง): กล่องจะ หลุดออกจาก Normal Flow ทันที (ไม่กินพื้นที่ Layout อีกต่อไป) และมันจะขยับตัวอ้างอิงกับ Element พ่อแม่ตัวแรกสุดที่มีค่า position ไม่ใช่ static (มักจะใช้คู่กับการตั้งค่า position: relative; ไว้ที่กล่องแม่) ถ้าไม่เจอพ่อแม่ที่มี position เลย มันจะอ้างอิงกับขอบของหน้าจอ (body/html)
  4. fixed (อ้างอิงหน้าจอ Viewport): หลุดออกจาก Normal Flow เช่นกัน แต่มันจะอ้างอิงกับ "หน้าต่างเบราว์เซอร์ (Viewport)" เสมอ ไม่ว่าผู้ใช้จะเลื่อน (Scroll) หน้าเว็บขึ้นลงไปไกลแค่ไหน กล่องนี้ก็จะติดหนึบอยู่ที่พิกัดเดิมบนหน้าจอ (นิยมใช้ทำแถบเมนูด้านบนที่เลื่อนตามลงมา หรือปุ่มแชท/ปุ่มกลับขึ้นบนสุดที่มุมขวาล่าง)
  5. sticky (ลูกผสมระหว่าง relative และ fixed): ตอนแรกมันจะทำตัวเหมือน relative คือไหลไปตามหน้าเว็บปกติ แต่พอเราเลื่อนหน้าจอจนกรอบของมันไปชนกับพิกัดที่เราตั้งไว้ (เช่น top: 0;) มันจะเปลี่ยนร่างเป็น fixed และติดหนึบที่ขอบจอบนทันที (เหมาะสำหรับทำหัวข้อ Section ที่เวลาเลื่อนผ่านเนื้อหาแล้วยังคงแสดงอยู่)

3. z-index และ Stacking Context (มิติความลึกแกน Z)

เมื่อเราใช้ absolute, fixed, หรือ relative กล่องต่างๆ มักจะลอยมาทับซ้อนกันได้ (Overlap) ปัญหาคือ กล่องไหนจะอยู่บน กล่องไหนจะอยู่ล่าง?

เบราว์เซอร์มีกฎพื้นฐานว่า กล่องที่เขียนใน HTML ทีหลัง (อยู่ล่างๆ ของไฟล์) จะทับกล่องที่เขียนก่อนเสมอ แต่เราสามารถเปลี่ยนลำดับนี้ได้ด้วย Property z-index (นึกถึงกราฟ 3 มิติ แกน X แนวนอน, แกน Y แนวตั้ง, แกน Z คือแนวลึกเข้าหาตัวเรา)

z-index รับค่าเป็นตัวเลขจำนวนเต็ม (เช่น 1, 10, 999) ยิ่งตัวเลขมาก กล่องนั้นยิ่งลอยอยู่ด้านบนสุด ข้อควรระวัง: z-index จะทำงานก็ต่อเมื่อ Element นั้นมีค่า position เป็น relative, absolute, fixed หรือ sticky เท่านั้น (ใส่ใน static จะไม่มีผลใดๆ)

💡 Pro Tip: ปัญหา Stacking Context บางครั้งคุณอาจเจอเหตุการณ์ที่ตั้ง z-index: 9999; ให้ลูกกล่องนึงแล้ว แต่ก็ยังโดนกล่องอื่นที่มี z-index: 1; ทับอยู่ดี นั่นเป็นเพราะระบบ Stacking Context (บริบทการซ้อนทับ) เปรียบเสมือนแฟ้มเอกสาร ตัวเลข z-index จะแข่งกันเฉพาะใน "แฟ้ม" (Element แม่) เดียวกันเท่านั้น ถ้าแฟ้มแม่ของกล่องคุณมี z-index ต่ำกว่าแฟ้มของอีกกล่อง ต่อให้เอกสารลูกข้างในจะใส่ z-index เป็นล้าน ก็ไม่มีทางทะลุกรอบของแฟ้มแม่ขึ้นไปสู้กับคนอื่นได้!
