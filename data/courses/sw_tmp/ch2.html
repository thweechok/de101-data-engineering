1. พื้นฐานของ CSS (CSS Fundamentals)

CSS (Cascading Style Sheets) คือเวทมนตร์ที่เปลี่ยนเอกสาร HTML ธรรมดาๆ ให้กลายเป็นเว็บไซต์ที่สวยงามน่าดึงดูด กฎพื้นฐานของการเขียน CSS ประกอบด้วย 2 ส่วนหลักคือ Selector (ตัวระบุว่าเราต้องการเลือกตกแต่งองค์ประกอบไหน) และ Declaration Block (กลุ่มของคำสั่งสไตล์ที่จะนำไปใช้ ซึ่งครอบด้วยปีกกา {})

selector {
  property: value; /* Declaration */
  color: blue;
}

2. Selectors (การเลือก Element ที่ต้องการตกแต่ง)

การจะตกแต่งเว็บให้แม่นยำ คุณต้องรู้จักวิธีเลือก (Select) สิ่งที่ถูกต้อง CSS มี Selector หลายประเภทให้ใช้งาน:

  1. Type Selector (Tag Selector): เลือกตามชื่อแท็ก HTML โดยตรง จะมีผลกับทุกแท็กนั้นบนหน้าเว็บ 
    h1 { color: red; }
  2. Class Selector: เลือกตามชื่อ Class โดยใช้เครื่องหมายจุด . นำหน้า (นิยมใช้มากที่สุด) เพราะเราสามารถกำหนด Class ให้กับกี่ Element ก็ได้ และนำกลับมาใช้ซ้ำ (Reusable) ได้ง่าย 
    .btn-primary { background-color: blue; color: white; }
  3. ID Selector: เลือกตามชื่อ ID โดยใช้เครื่องหมายชาร์ป # นำหน้า (ควรมีอันเดียวในหน้าเว็บ) นิยมใช้สำหรับโครงสร้างหลักหรือจุดที่ต้องการระบุแบบเฉพาะเจาะจงขั้นสุด 
    #main-header { padding: 20px; }
  4. Attribute Selector: เลือก Element ที่มี Attribute ตามที่กำหนด 
    input[type="text"] { border: 1px solid gray; }

3. Specificity (ความเฉพาะเจาะจง - กฎการต่อสู้ของ CSS)

เมื่อคุณเขียน CSS โครงการใหญ่ๆ คุณมักจะเจอปัญหาว่า "ทำไมสไตล์ที่ฉันเขียนถึงไม่แสดงผล?" สาเหตุหลักมาจาก Specificity หากมีกฎ CSS หลายข้อพยายามเขียนทับ (Overwrite) สไตล์ของ Element เดียวกัน เบราว์เซอร์จะคำนวณคะแนนความเฉพาะเจาะจงเพื่อตัดสินว่ากฎไหนจะเป็นผู้ชนะ

ลำดับความสำคัญจากมากไปน้อย (คะแนนคร่าวๆ):

  • Inline Style (เขียนแทรกในแท็ก HTML: style="...") -> ชนะทุกอย่าง (ยกเว้น !important)
  • ID Selector (#id) -> คะแนนสูงมาก
  • Class Selector, Pseudo-class (.class, :hover) -> คะแนนปานกลาง
  • Type Selector (h1, p) -> คะแนนต่ำสุด

หากคะแนนเท่ากัน เบราว์เซอร์จะเลือกกฎที่ เขียนอยู่ล่างสุด (ลำดับสุดท้าย) ตามความหมายของคำว่า "Cascading" (น้ำตกที่ไหลจากบนลงล่าง)

💡 Pro Tip: การใช้ !important คุณสามารถเติม !important ไว้หลัง value (เช่น color: red !important;) เพื่อบังคับให้กฎนี้ชนะทุกสิ่ง อย่างไรก็ตาม ควรหลีกเลี่ยงการใช้ !important ให้มากที่สุด เพราะมันจะทำลายระบบ Cascading และทำให้โค้ดแก้บัค (Debug) ยากในอนาคต ให้ใช้วิธีเพิ่ม Class หรือแก้ไข Specificity ให้ถูกต้องจะดีกว่า

4. Inheritance (การสืบทอดคุณสมบัติ)

ใน CSS Property (คุณสมบัติ) บางอย่างจะถูก สืบทอด (Inherit) จาก Element พ่อแม่ (Parent) ไปยัง Element ลูก (Child) โดยอัตโนมัติ คุณสมบัติที่มักจะถูกสืบทอดจะเป็นพวก Typography (ตัวอักษร) เช่น color, font-family, font-size, line-height

ส่วน Property ที่เกี่ยวกับการจัดวาง เช่น margin, padding, border, background จะ ไม่ถูกสืบทอด เพราะมันคงจะวุ่นวายมากถ้าคุณใส่ขอบ (Border) ให้คอนเทนเนอร์หลัก แล้วปุ่มทุกปุ่มข้างในได้ขอบนั้นไปด้วย

5. Colors และ Typography (สีสันและตัวอักษร)

การกำหนดสี (Colors) ใน CSS สามารถทำได้หลายวิธี:

  • Keyword: ใช้ชื่อสีตรงๆ เช่น red, blue, transparent (โปร่งใส)
  • HEX Code: รหัสสีฐาน 16 (ขึ้นต้นด้วย #) เช่น #FF0000 นิยมใช้มากที่สุดในดีไซน์
  • RGB / RGBA: การผสมสีแดง เขียว น้ำเงิน (0-255) โดยมีตัว A (Alpha) ควบคุมความโปร่งแสง (Opacity) เช่น rgba(0, 0, 0, 0.5) คือสีดำโปร่งแสง 50%
  • HSL: Hue, Saturation, Lightness การกำหนดสีตามวงล้อสี เหมาะสำหรับการทำไล่เฉดสี

การจัดการตัวอักษร (Typography) Properties ที่สำคัญที่สุดมีดังนี้:

  • font-family: กำหนดตระกูลฟอนต์ ควรมีฟอนต์สำรอง (Fallback) เสมอ เช่น font-family: 'Helvetica', Arial, sans-serif;
  • font-size: กำหนดขนาด สามารถใช้หน่วย px, em, หรือ rem
  • font-weight: ความหนาของฟอนต์ ค่าเริ่มต้นคือ 400 ส่วนตัวหนา (bold) คือ 700
  • line-height: ความสูงของบรรทัด (ระยะห่างระหว่างบรรทัด) เพื่อความอ่านง่าย ควรตั้งค่าไว้ที่ประมาณ 1.5 หรือ 1.6 สำหรับข้อความเนื้อหาทั่วไป
  • text-align: จัดตำแหน่งข้อความ (left, center, right, justify)
