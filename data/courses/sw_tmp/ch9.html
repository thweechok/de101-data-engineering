1. เพิ่มชีวิตชีวาและความนุ่มนวลด้วย Transitions

ในโลกความเป็นจริง เมื่อเราเปิดประตู หรือกดสวิตช์ไฟ มันไม่ได้เกิดขึ้นในพริบตา 0 วินาที มันต้องใช้เวลา แต่ในโลกของเว็บยุคเก่า การเปลี่ยนแปลงสไตล์อย่างกระทันหัน (เช่น เอาเมาส์ไปชี้ปุ่มแล้วสีเปลี่ยนสลับทันที) มักทำให้ผู้ใช้รู้สึกแข็งกระด้าง CSS Transitions คือเครื่องมือที่เข้ามาแก้ปัญหานี้ โดยช่วยให้การเปลี่ยนแปลงค่าระหว่าง 2 สถานะ (State) เกิดขึ้นอย่างค่อยเป็นค่อยไป (Interpolation) ทำให้เว็บไซต์ดูพรีเมียมและเป็นธรรมชาติขึ้นมาก

การใช้ Transition จะต้องประกอบด้วยส่วนสำคัญ 4 อย่าง (มักเขียนรวบยอดแบบ Shorthand):

  1. transition-property: เลือกสิ่งที่จะให้ทำแอนิเมชัน เช่น background-color, transform หรือ all (เปลี่ยนทุกอย่าง)
  2. transition-duration: ระยะเวลาที่ใช้ตั้งแต่ต้นจนจบ เช่น 0.3s (0.3 วินาที) หรือ 300ms
  3. transition-timing-function: อัตราเร่งของเวลา (เช่น ease-in เริ่มต้นช้าแล้วพุ่งเร็ว, ease-out เริ่มเร็วแล้วค่อยๆ เบรก, linear ความเร็วคงที่)
  4. transition-delay: รอกี่วินาทีก่อนที่จะเริ่มแสดงผล (หน่วงเวลา)
.btn {
  background-color: blue;
  /* แบบย่อ: property | duration | timing-function | delay */
  transition: background-color 0.3s ease-in-out;
}
.btn:hover {
  background-color: red; /* เมื่อเมาส์ชี้ สีจะค่อยๆ ละลายเปลี่ยนเป็นแดงในเวลา 0.3 วินาที */
}

2. ศาสตร์แห่งการแปลงโฉมเรขาคณิต (CSS Transform)

transform คือ Property ที่ยอดเยี่ยมที่สุดสำหรับการขยับ, ย่อ, ขยาย, หมุน หรือบิด Element จุดเด่นที่ทำให้มันเหนือกว่าการเปลี่ยนค่าแบบอื่นคือ มันไม่ส่งผลกระทบต่อ Layout (Normal Flow) ของคนอื่นรอบข้าง (ถ้าคุณขยายขนาดกล่องด้วย transform กล่องข้างๆ จะไม่โดนเบียด) และที่สำคัญ มันถูกประมวลผลโดยตรงด้วยการ์ดจอ (Hardware Accelerated) ทำให้การแสดงผลลื่นไหลมาก (ถึง 60fps) ฟังก์ชันที่ใช้บ่อยได้แก่:

  • translate(x, y): เลื่อนตำแหน่งไปตามแกน X และ Y (เช่น ทำแอนิเมชันปุ่มเด้ง, หรือ Modal เลื่อนหล่นลงมาจากฟ้า)
  • scale(x, y): ย่อหรือขยายขนาดสัดส่วน (เช่น scale(1.1) คือการขยายขึ้น 10% จากเดิม)
  • rotate(deg): หมุนเป็นองศา (เช่น rotate(45deg), rotate(1turn))

3. @keyframes: ก้าวข้ามขีดจำกัดด้วย Animations

แม้ Transitions จะดี แต่ก็มีข้อจำกัดตรงที่ "มันต้องการเหตุการณ์มากระตุ้น" (เช่น ต้องมี :hover) และมันขยับได้แค่ 2 สเต็ป คือจาก A ไป B แต่ถ้าเราต้องการทำแอนิเมชันที่ซับซ้อน, เล่นเองอัตโนมัติตั้งแต่โหลดเว็บเสร็จ, และมีหลายจังหวะ เราจะต้องขยับไปใช้ CSS Animations คู่กับ @keyframes

@keyframes คือการกำหนดจุดย่อยๆ ของเวลา (Timeline) เราสามารถบอกได้อย่างละเอียดว่าที่ 0% (เริ่มต้น) ให้ทำอะไร ที่ 50% ให้เปลี่ยนเป็นอะไร และ 100% ให้จบอย่างไร

/* 1. สร้าง Keyframes (บทบาทสมมติ) ตั้งชื่อว่า 'bounce' */
@keyframes bounce {
  0%   { transform: translateY(0); }
  50%  { transform: translateY(-20px); } /* จังหวะกลาง: กระโดดลอยขึ้นไป 20px */
  100% { transform: translateY(0); } /* จบ: ตกกลับลงมาที่เดิม */
}

/* 2. เรียกใช้งานด้วย property 'animation' */
.loading-ball {
  /* ชื่อบทบาท | ระยะเวลา | อัตราเร่ง | หน่วงเวลา | จำนวนรอบ (infinite คือวนลูปตลอดไป) */
  animation: bounce 1s ease-in-out infinite; 
}
💡 Performance Tip: เคล็ดลับความลื่นไหล กฎเหล็กของการทำแอนิเมชันบนเว็บที่ดี ห้าม! ไปทำ transition หรือ animate กับ property ที่เปลี่ยนขนาดกล่องหรือตำแหน่งจริงๆ เช่น width, height, margin, top, left เด็ดขาด เพราะการเปลี่ยนค่าเหล่านี้จะบังคับให้เบราว์เซอร์คำนวณ Layout ของเว็บใหม่ทั้งหมดในทุกๆ เฟรม ทำให้หน้าเว็บกระตุก (Jank) อย่างรุนแรง
ให้ใช้แค่ transform (translate, scale, rotate) และ opacity (ความโปร่งใส) เท่านั้น! เพราะ 2 ตัวนี้เป็น Compositor Properties ที่เบราว์เซอร์ส่งไปให้ GPU วาดภาพให้โดยตรง
