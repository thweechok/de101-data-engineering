1. ยุคทองของ CSS กับฟีเจอร์ระดับ "เกมเชนเจอร์"

CSS ไม่เคยหยุดพัฒนาและเติบโต ในช่วงไม่กี่ปีที่ผ่านมา W3C ได้ปล่อยฟีเจอร์ระดับ "เกมเชนเจอร์" (Game Changer) เข้ามาในมาตรฐานเบราว์เซอร์มากมาย ทำให้การเขียน CSS สมัยใหม่สะดวก รวดเร็ว และทรงพลังเทียบเท่าหรือเหนือกว่าเครื่องมืออย่าง SASS โดยที่เราไม่จำเป็นต้องติดตั้งโปรแกรมคอมไพล์เพิ่มเติมเลย

2. Native CSS Nesting (การเขียนซ้อนทับตามลำดับชั้น)

ปัญหาคลาสสิกของคนเขียน CSS เพียวๆ คือ การเขียน Selector ลูกหลานที่ต้องเขียนคลาสของแม่ซ้ำๆ จนโค้ดยาวเฟื้อยและอ่านยาก แต่ด้วยมาตรฐาน Native CSS Nesting เราสามารถนำ Selector ของลูก ไปเขียนซ้อน (Nest) ไว้ข้างในวงเล็บปีกกาของตัวแม่ได้โดยตรงเลย!

/* สมัยก่อน (อดีตอันแสนเหนื่อยล้า) */
.card { padding: 20px; background: white; }
.card .title { color: blue; font-size: 24px; }
.card .title:hover { text-decoration: underline; }

/* ยุค Modern CSS (Nesting - เหมือน SASS เป๊ะ!) */
.card {
  padding: 20px;
  background: white;
  
  .title {
    color: blue;
    font-size: 24px;
    
    /* เครื่องหมาย & (Ampersand) หมายถึงตัวมันเอง (.title) */
    &:hover {
      text-decoration: underline;
    }
  }
}

3. พลังวิเศษของ :has() (The Parent Selector)

สิ่งที่ Developer ทั่วโลกเรียกร้องและเฝ้ารอกันมาเป็นสิบปีคือ "การเลือก Element พ่อแม่ โดยอ้างอิงจากเงื่อนไขของ Element ลูก" ซึ่งในอดีตเราต้องพึ่งพา JavaScript ล้วนๆ แต่ตอนนี้ความฝันเป็นจริงแล้วด้วย Pseudo-class :has()

ตัวอย่าง: สมมติว่ามีกล่องการ์ด (Card) ถ้าภายในการ์ดนี้ "มี" รูปภาพ (img) อยู่ข้างใน เราจะให้การ์ดใบนั้นมีพื้นหลังสีดำ แต่ถ้าการ์ดไหนมีแต่ตัวหนังสือ ก็ให้ใช้สีขาวปกติ

.card { background: white; color: black; }

/* แปลความหมาย: ถ้า .card ใบไหน :has(มีแท็ก img อยู่ข้างใน) ให้ทำงานตามกฎนี้ */
.card:has(img) { 
  background: black; 
  color: white; 
}

นอกจากนี้ยังมี :is() และ :where() ที่ช่วยรวมกลุ่ม Selector ยาวๆ หรือชื่อแท็กซ้ำๆ ให้สั้นลงเหลือเพียงบรรทัดเดียว เช่น แทนที่จะเขียน h1, h2, h3, h4, h5, h6 { color: red; } เราสามารถรวบเป็น :is(h1, h2, h3, h4, h5, h6) { color: red; } ได้เลย

4. Scroll-Snap (การปัดหน้าจอแบบล็อคตำแหน่งสไตล์แอปมือถือ)

เวลาทำแกลเลอรีรูปภาพแนวนอน หรือ Carousel (สไลเดอร์) ปกติเราต้องเขียน JavaScript ปวดหัว หรือลงไลบรารีเพื่อล็อคให้รูปมาหยุดแสดงตรงกลางจอพอดี แต่ตอนนี้เรามี CSS scroll-snap ที่จัดการพฤติกรรมนี้ได้จบใน CSS เพียวๆ

.carousel-container {
  display: flex;
  overflow-x: auto;
  /* เปิดใช้งานการล็อคสแนปแกน X แบบบังคับ (mandatory) */
  scroll-snap-type: x mandatory; 
}
.carousel-item {
  flex: 0 0 100%; /* ให้กว้างเต็มจอ */
  /* เมื่อเลื่อนแล้วปล่อยนิ้ว กล่องนี้จะเด้งมาล็อคที่กึ่งกลางจอเสมอ */
  scroll-snap-align: center; 
}

5. ฟีเจอร์ที่น่าสนใจอื่นๆ (Aspect-Ratio และ Backdrop-Filter)

  • aspect-ratio: การล็อคสัดส่วนของกล่อง สมัยก่อนจะฝังวิดีโอ Youtube ให้สัดส่วน 16:9 แบบ Responsive ต้องใช้ "Padding Hack" ที่ซับซ้อน ปัจจุบันเขียนแค่ aspect-ratio: 16 / 9; กล่องจะยืดหดตามความกว้าง แต่รักษาความสูงให้เป็นสัดส่วนเดิมเสมอ (ไม่บิดเบี้ยว)
  • backdrop-filter: เอฟเฟกต์เบลอพื้นหลังที่ถูกทับ (Glassmorphism หรือสไตล์กระจกฝ้าแบบ Apple) เพียงแค่ใช้ backdrop-filter: blur(10px); ควบคู่กับพื้นหลังแบบโปร่งแสง background: rgba(255,255,255,0.5); คุณก็จะได้หน้าต่างใสๆ เบลอๆ แสนสวยงามทันที
💡 ข้อควรระวัง: การเช็ค Browser Support แม้ฟีเจอร์ยุคใหม่จะยอดเยี่ยมแค่ไหน แต่เบราว์เซอร์รุ่นเก่าๆ (ที่ไม่อัปเดต) อาจจะยังไม่รู้จักคำสั่งเหล่านี้ คุณสามารถป้องกันเว็บพังได้โดยใช้คำสั่ง @supports (property: value) ใน CSS เพื่อเช็คระบบก่อนว่า "เบราว์เซอร์นี้รู้จักคำสั่งใหม่นี้ไหม?" ถ้ารู้จักถึงจะรันโค้ดข้างใน เป็นเทคนิคที่เรียกว่า Feature Query ที่โปรเฟสชันนัลใช้กัน
