1. Responsive Design คืออะไร?

ในยุคอดีตที่ผู้คนเข้าเว็บไซต์ผ่านทางหน้าจอคอมพิวเตอร์แบบตั้งโต๊ะ (Desktop) เพียงอย่างเดียว การกำหนดขนาดเลย์เอาต์ทุกอย่างเป็น Pixel (px) แบบตายตัวก็เพียงพอแล้ว แต่ในปัจจุบัน โลกเปลี่ยนไป ผู้คนเข้าถึงอินเทอร์เน็ตผ่านอุปกรณ์ที่หลากหลาย ตั้งแต่สมาร์ทวอทช์หน้าจอจิ๋ว โทรศัพท์มือถือ แท็บเล็ต ไปจนถึงทีวีจอแบนขนาด 80 นิ้ว Responsive Web Design (RWD) จึงถือกำเนิดขึ้น เพื่อให้เว็บไซต์สามารถ "ปรับตัว" และแสดงผลได้อย่างสวยงาม ใช้งานง่าย อ่านสะดวก ในทุกขนาดหน้าจอ โดยใช้โค้ด HTML ชุดเดียวกัน

2. หัวใจหลัก: Media Queries (@media)

พระเอกตลอดกาลของ Responsive Design คือคำสั่งวิเศษที่เรียกว่า Media Query ซึ่งทำหน้าที่เหมือน "สวิตช์เปิด-ปิด CSS" โดยมันจะตรวจสอบขนาดหน้าจอของผู้ใช้ (Viewport Width) และจะบังคับใช้กฎ CSS ที่อยู่ข้างในปีกกา ก็ต่อเมื่อเงื่อนไขขนาดหน้าจอนั้นเป็นจริง

/* CSS ปกติ (มีผลกับทุกหน้าจอเป็นค่าตั้งต้น) */
.container {
  width: 100%;
}

/* เมื่อหน้าจอมีความกว้าง "อย่างน้อย" 768px (เช่น Tablet หรือ Desktop) */
@media (min-width: 768px) {
  /* โค้ดในนี้จะทำงานและทับโค้ดด้านบน เมื่อจอใหญ่พอ */
  .container {
    width: 50%;
  }
}

3. แนวคิด Mobile-First Approach

การเขียน Responsive ที่ได้รับการยอมรับว่าเป็น Best Practice ในปัจจุบันคือการเขียนแบบ Mobile-First ซึ่งหมายถึงการเขียน CSS เริ่มต้นให้เป็นดีไซน์ของ "มือถือ" ก่อน (ไม่ใส่ Media Query) จากนั้นค่อยใช้ @media (min-width: ...) เพื่อค่อยๆ ขยายหรือปรับเพิ่มความซับซ้อนของเลย์เอาต์เมื่อหน้าจอใหญ่ขึ้นทีละระดับ

ข้อดีของ Mobile-First คือ โค้ดจะอ่านง่าย เป็นลำดับตรรกะที่ถูกต้อง และที่สำคัญคือ เบราว์เซอร์มือถือ (ซึ่งประมวลผลช้ากว่าคอมพิวเตอร์และใช้อินเทอร์เน็ตจำกัด) จะไม่ต้องมานั่งโหลด CSS ของ Desktop ที่ซับซ้อนมาแล้วค่อยแก้กลับเป็นของมือถือ

4. Viewport Units และ Relative Units (ความยาวแบบยืดหยุ่น)

การทำ Responsive ที่ดี จะพึ่งพาหน่วยที่เป็นตัวเลขตายตัว (Absolute unit) อย่าง px ให้น้อยลง และหันมาใช้หน่วยที่ยืดหยุ่น (Relative unit) แทน:

  • vw (Viewport Width) และ vh (Viewport Height): หน่วยอ้างอิงความกว้าง/ความสูงของหน้าจอโดยตรง (1vw = 1% ของความกว้างหน้าจอ) เหมาะมากสำหรับทำแบนเนอร์หรือหน้า Hero section ให้เต็มจอเป๊ะๆ (height: 100vh)
  • % (Percentage): อ้างอิงจากขนาดของ Element พ่อแม่
  • em และ rem (หน่วยอ้างอิงฟอนต์):
    • rem (Root em): อ้างอิงจากฟอนต์ตั้งต้นของเบราว์เซอร์ (มักจะเป็น 16px) นิยมใช้กำหนด font-size เพื่อให้ผู้ใช้สามารถปรับขนาดตัวอักษรเพื่อความสะดวกในการอ่าน (Accessibility) ได้
    • em: อ้างอิงจากฟอนต์ของ Element พ่อแม่ตัวเอง นิยมใช้ตั้งค่า padding หรือ margin ที่ควรขยาย/หด ตามขนาดตัวอักษรของกล่องนั้นๆ
💡 Pro Tip: Fluid Typography ด้วยฟังก์ชัน clamp() เคล็ดลับระดับโปรในการทำ Responsive ให้ตัวอักษรยืดหดเนียนๆ อย่างต่อเนื่อง โดยไม่ต้องเขียน Media Query ซ้ำๆ หลายบรรทัด คือการใช้ฟังก์ชัน clamp()
ตัวอย่าง: font-size: clamp(1rem, 2vw + 1rem, 3rem);
ความหมาย: ฟอนต์จะเล็กสุดที่ 1rem, ขนาดปกติจะไหลขยายตามความกว้างจอ (2vw + 1rem), แต่จะห้ามใหญ่เกิน 3rem เด็ดขาด! นี่คือความสมบูรณ์แบบของการออกแบบตัวอักษร
