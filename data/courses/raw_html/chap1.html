โครงสร้างพื้นฐานของเอกสาร HTML

ไฟล์ HTML ทุกไฟล์ต้องมีโครงสร้างพื้นฐานที่เบราว์เซอร์คาดหวัง เพื่อให้สามารถประมวลผลและแสดงผลได้อย่างถูกต้อง ลองดูโครงสร้างพื้นฐานนี้ (หรือที่เรียกว่า Boilerplate):

<!DOCTYPE html>
<html lang="th">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First Webpage</title>
</head>
<body>
    <!-- เนื้อหาเว็บจะอยู่ที่นี่ -->
</body>
</html>
  • <!DOCTYPE html>: ประกาศให้เบราว์เซอร์รู้ว่านี่คือ HTML5
  • <html>: Root element ครอบทุกสิ่งในหน้าเว็บ
  • <head>: ส่วนที่เก็บ Meta data เช่น title, charset และลิงก์ไปยังไฟล์ CSS
  • <body>: ส่วนที่เก็บเนื้อหาทั้งหมดที่ผู้ใช้มองเห็น

Tags, Elements และ Attributes

HTML Tag คือคำสั่งที่ใช้บอกเบราว์เซอร์ว่าเนื้อหาส่วนนั้นคืออะไร มักมาเป็นคู่คือ Opening Tag และ Closing Tag (เช่น <p>...</p>) แต่บาง Tag ก็ไม่มีคู่ เรียกว่า Self-closing tags (เช่น <img />, <br />)

HTML Element คือส่วนประกอบทั้งหมดตั้งแต่ Opening Tag, Content ไปจนถึง Closing Tag

Attributes คือคุณสมบัติพิเศษที่ใส่เพิ่มใน Opening Tag เพื่อให้ข้อมูลเพิ่มเติม ตัวอย่างเช่น:

<a href="https://example.com" target="_blank" class="my-link">Click me!</a>

ในตัวอย่างนี้ href, target, และ class คือ Attributes

Semantic HTML: เขียนโค้ดให้มีความหมาย

ในยุคก่อน ผู้คนมักใช้ <div> ครอบทุกอย่าง (Div Soup) แต่ใน HTML5 เรามี Semantic Tags ที่สื่อความหมายชัดเจนว่าแต่ละส่วนคืออะไร ซึ่งดีต่อ SEO (Search Engine Optimization) และ Accessibility (การเข้าถึงสำหรับผู้พิการ):

  1. <header>: ส่วนหัวของหน้าเว็บ มักมีโลโก้หรือเมนูนำทาง
  2. <nav>: ส่วน Navigation หรือเมนูหลักของเว็บ
  3. <main>: เนื้อหาหลักของหน้า (ควรมีแค่ตัวเดียวต่อหน้า)
  4. <article>: เนื้อหาที่สมบูรณ์ในตัวเอง สามารถนำไปใช้ที่อื่นได้ (เช่น บทความบล็อก)
  5. <section>: แบ่งเนื้อหาเป็นส่วนๆ ตามหัวเรื่อง
  6. <aside>: เนื้อหาเสริม ด้านข้าง หรือโฆษณา
  7. <footer>: ส่วนท้ายของเว็บ มักมีลิขสิทธิ์หรือลิงก์เสริม

Forms และ Inputs: การรับข้อมูลจากผู้ใช้

ฟอร์ม (Form) เป็นหัวใจสำคัญของการปฏิสัมพันธ์บนเว็บ (Interactive) เช่น หน้า Login หรือ สมัครสมาชิก โครงสร้างฟอร์มพื้นฐานประกอบด้วย <form> และ input ประเภทต่างๆ:

<form action="/submit" method="POST">
    <label for="username">ชื่อผู้ใช้:</label>
    <input type="text" id="username" name="user" required>
    
    <label for="email">อีเมล:</label>
    <input type="email" id="email" name="email" required>
    
    <button type="submit">ลงทะเบียน</button>
</form>

ประเภทของ Input ที่ควรรู้จัก: text, password, email, number, checkbox, radio, date, และ file

💡 Pro Tip: การใช้ Attribute for ใน <label> ให้ตรงกับ id ของ <input> จะทำให้ผู้ใช้สามารถคลิกที่ข้อความ label เพื่อเลือก input นั้นได้เลย เพิ่มประสบการณ์ผู้ใช้ (UX) ได้อย่างมาก โดยเฉพาะในโทรศัพท์มือถือ!
