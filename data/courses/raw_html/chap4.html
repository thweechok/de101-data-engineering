ศาสตร์แห่งการจัดวาง: Positioning ใน CSS

ตามปกติแล้ว เบราว์เซอร์จะจัดเรียงเนื้อหาจากบนลงล่าง ซ้ายไปขวา นี่เรียกว่า Normal Document Flow แต่บางครั้งเราต้องการบังคับให้ Element ไปอยู่ตำแหน่งเฉพาะ เช่น แถบเมนูด้านบนที่เลื่อนตามลงมา หรือปุ่มแชทมุมขวาล่าง นั่นคือหน้าที่ของ property ที่ชื่อ position

ค่าต่างๆ ของ Position

  1. Static (ค่าเริ่มต้น): กล่องวางเรียงตาม Normal Flow ปกติ ไม่สามารถใช้คำสั่ง top, right, bottom, left หรือ z-index เพื่อขยับมันได้
  2. Relative: ยังคงอยู่ใน Normal Flow เหมือน static แต่คุณสามารถใช้ top, right, bottom, left เพื่อ "ขยับมันออกจากตำแหน่งเดิม" ได้ (โดยพื้นที่เดิมของมันจะยังคงถูกจองเอาไว้ ไม่หายไปไหน) มักใช้สำหรับทำเป็น "จุดอ้างอิง" (Container) ให้กับ Absolute position
  3. Absolute: หลุดออกจาก Normal Flow โดยสิ้นเชิง (Element อื่นจะขยับขึ้นมาแทนที่พื้นที่ของมัน) มันจะอ้างอิงตำแหน่งและจัดวางตัวเองเทียบกับ "บรรพบุรุษที่ใกล้ที่สุดที่มี position ไม่ใช่ static" (ถ้าหาไม่เจอ จะอ้างอิงกับ <html> หรือ Viewport แทน)
  4. Fixed: คล้ายกับ Absolute คือหลุดจาก Flow แต่จะอ้างอิงตำแหน่งกับ Viewport (หน้าต่างเบราว์เซอร์) เสมอ ไม่ว่าคุณจะ Scroll หน้าเว็บลงไปแค่ไหน มันก็จะยังคงติดอยู่ที่ตำแหน่งเดิมบนหน้าจอ (เช่น Navigation bar แบบติดหนึบ)
  5. Sticky: ลูกผสมระหว่าง Relative และ Fixed ในตอนแรกมันจะประพฤติตัวเหมือน Relative ไปเรื่อยๆ จนกระทั่งคุณ Scroll หน้าจอมาถึงจุดที่กำหนด (เช่น top: 0;) มันจะกลายร่างเป็น Fixed และติดหนึบอยู่ตรงนั้นจนกว่ากล่องแม่ของมัน (Container) จะถูก Scroll ผ่านไป

ตัวอย่างการใช้งาน Relative + Absolute

เทคนิคที่ใช้บ่อยที่สุดในการวางซ้อนองค์ประกอบ (เช่น การทำปุ่ม "X" ปิดรูปที่มุมขวาบนของรูปภาพ) คือการเซ็ตกล่องครอบ (Parent) เป็น Relative และเซ็ตของข้างใน (Child) เป็น Absolute

.card-container {
    position: relative; /* เป็นจุดอ้างอิง */
    width: 300px;
    height: 400px;
    background: #eee;
}

.close-btn {
    position: absolute;
    top: 10px;    /* ห่างจากขอบบนของ .card-container 10px */
    right: 10px;  /* ห่างจากขอบขวาของ .card-container 10px */
}

Z-Index และ Stacking Context (ลำดับการซ้อนทับ)

เมื่อเราใช้ Position ขยับ Element ออกจาก Flow มักจะเกิดการ "ซ้อนทับกัน" (Overlapping) เราสามารถควบคุมได้ว่าใครจะอยู่บน ใครจะอยู่ล่าง ด้วย z-index

  • ค่า z-index เป็นตัวเลข (เช่น -1, 0, 1, 10, 9999) ยิ่งค่ามาก ยิ่งอยู่ด้านบน
  • ข้อควรระวัง: z-index จะทำงานก็ต่อเมื่อ Element นั้นมี position ที่ ไม่ใช่ static (เช่น relative, absolute, fixed, sticky) เท่านั้น!
  • Stacking Context: z-index จะเปรียบเทียบกันภายในบริบทเดียวกัน (กล่องแม่เดียวกัน) ถ้ากล่องแม่มี z-index ต่ำ ต่อให้ลูกมี z-index เป็นแสน ก็ไม่สามารถทะลุขึ้นไปอยู่เหนือกล่องอื่นที่มี z-index สูงกว่าได้
💡 Pro Tip: อย่าสร้างปัญหา "Z-index War" (การใส่ z-index: 99999 มั่วไปหมดเพื่อแก้ปัญหาเฉพาะหน้า) ให้วางแผน Stacking Context ตั้งแต่ต้น ตัวอย่างระบบสเกลที่ดี: Modal=1000, Dropdown=500, Sticky Header=100, Base=1 เป็นต้น
