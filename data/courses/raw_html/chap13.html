เวลาแสดงฝีมือ! (Time to Build)

การเรียนรู้โค้ดจากการอ่านและดูวิดีโอนั้นเทียบไม่ได้เลยกับการ ลงมือทำโปรเจกต์จริง ในบทสุดท้ายนี้ เราจะนำความรู้ทั้งหมดตั้งแต่ HTML Semantic Tags, Box Model, Flexbox/Grid, Responsive Design ไปจนถึง CSS Animations มาสร้าง "เว็บไซต์ Portfolio ของคุณเอง"!

โครงสร้างหน้าเว็บ (The Architecture)

เว็บไซต์ Portfolio ที่ดีควรดูสะอาด เป็นมืออาชีพ และเล่าเรื่องราวของคุณได้ โครงสร้างพื้นฐานที่เราจะสร้าง (Single Page Application) ประกอบด้วย 4 ส่วนหลัก:

  1. Header & Navigation: แถบเมนูด้านบนแบบ Sticky ขยับตามเมื่อ Scroll มีปุ่มแฮมเบอร์เกอร์ (Hamburger menu) สำหรับมือถือ
  2. Hero Section: ส่วนแรกที่ผู้ใช้เห็นเมื่อโหลดหน้าเว็บ ต้องปัง! ประกอบด้วยรูปภาพของคุณ, ตำแหน่งงาน (เช่น Frontend Developer), และปุ่ม Call to Action (CTA) เช่น "ดูผลงาน" หรือ "โหลด Resume"
  3. Projects / Work Experience: ส่วนแสดงผลงาน จัดวางเลย์เอาต์ด้วย CSS Grid แบบ 3 คอลัมน์ (Desktop) หดเหลือ 1 คอลัมน์ (Mobile) เพิ่มเอฟเฟกต์ :hover ให้การ์ดเด้งขึ้นมาเล็กน้อย (Transform & Transition)
  4. Contact & Footer: ฟอร์มติดต่อ (HTML Forms) และลิงก์ไปยัง GitHub, LinkedIn พร้อมลิขสิทธิ์

เทคนิคที่นำมาประยุกต์ใช้ในโปรเจกต์

  • ใช้ CSS Variables (Custom Properties) ในการกำหนด Color Palette ตั้งแต่แรก ដើម្បីให้พร้อมสำหรับฟีเจอร์ Dark Mode Toggle ในอนาคต
  • กำหนด scroll-behavior: smooth; ในแท็ก html เพื่อให้เวลาคลิกเมนู แล้วหน้าเว็บค่อยๆ สไลด์ลงมาอย่างนุ่มนวล (ไม่ต้องพึ่ง JavaScript!)
  • ใช้ Clamp() สำหรับหัวข้อ (H1) ใน Hero Section เพื่อให้ขนาดฟอนต์ย่อขยายตามขนาดหน้าจออัตโนมัติ (Fluid Typography)
  • ใช้ Semantic HTML (เช่น <nav>, <section id="projects">, <article>, <footer>) เพื่อให้หน้าเว็บรองรับ Accessibility และ SEO แย่งอันดับใน Google

การนำขึ้นใช้งานจริง (Deployment)

เขียนโค้ดเสร็จแล้ว ถ้าอยู่แค่ในคอมพิวเตอร์ของคุณก็คงไม่มีใครเห็น! ขั้นตอนสุดท้ายคือการนำเว็บขึ้น Production (Live URL) ซึ่งปัจจุบันมีบริการโฮสติ้งฟรี (Free Tier) ที่ดีมากสำหรับเว็บ Static (HTML/CSS/JS):

  • GitHub Pages: คลาสสิกและทรงพลังที่สุดในวงการ เพียงแค่คุณอัพโหลดโค้ดขึ้น Repository ใน GitHub และเปิดใช้งาน Pages คุณก็จะได้ลิงก์ username.github.io/repo-name ใช้งานฟรีตลอดกาล
  • Vercel หรือ Netlify: แพลตฟอร์มยุคใหม่ที่ได้รับความนิยมสูงสุด ใช้งานง่ายมาก (Drag & Drop โฟลเดอร์ได้เลย) หรือจะเชื่อมต่อกับ GitHub เพื่อทำ CI/CD (เวลาคุณแก้โค้ดและ Push ขึ้น GitHub เว็บไซต์จะอัพเดตอัตโนมัติในไม่กี่วินาที)
💡 บทสรุป: ยินดีด้วย! คุณได้เดินทางจาก "Zero to Hero" ในโลกของ HTML & CSS แล้ว จากนี้ไป การเดินทางของคุณเพิ่งจะเริ่มต้น ขอให้สนุกกับการเขียนโค้ด และอย่าลืมว่า "นักพัฒนาที่เก่งที่สุด ไม่ใช่คนที่จำโค้ดได้ทุกบรรทัด แต่คือคนที่รู้จักวิธีค้นหา (Google/StackOverflow) และประยุกต์ใช้ได้อย่างสร้างสรรค์" ลุยเลย! 🚀
