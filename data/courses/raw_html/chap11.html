Web Accessibility (a11y) คืออะไร?

คำว่า Accessibility (มักย่อว่า a11y เพราะมี 11 ตัวอักษรระหว่าง a กับ y) หมายถึง "การเข้าถึงได้" หรือการออกแบบและพัฒนาเว็บไซต์ให้ ทุกคน สามารถใช้งานได้ ไม่ว่าจะเป็นผู้ที่มีสายตาปกติ, ผู้ที่มีความบกพร่องทางการมองเห็น (ต้องใช้โปรแกรม Screen Reader อ่านหน้าเว็บให้ฟัง), ผู้ที่ไม่สามารถใช้เมาส์ได้ (ต้องใช้คีย์บอร์ดอย่างเดียว), หรือผู้ที่มีภาวะตาบอดสี

การทำ a11y ไม่ได้เป็นเพียงแค่ "ความรับผิดชอบต่อสังคม" แต่ในหลายประเทศมีกฎหมายบังคับ และ Google ก็ให้คะแนน SEO กับเว็บไซต์ที่รองรับ a11y ดีกว่าด้วย!

Semantic HTML คือก้าวแรกที่ดีที่สุด

อย่างที่เราได้เรียนไปในบทที่ 1 การใช้ Semantic Tags เช่น <button>, <nav>, หรือ <main> แทนการใช้ <div> ล้วนๆ คือการทำ a11y ที่ดีที่สุด เบราว์เซอร์และ Screen Reader จะรู้ทันทีว่าปุ่มคือปุ่ม และรองรับการกดด้วยปุ่ม Enter/Spacebar บนคีย์บอร์ดโดยอัตโนมัติ

Alt Text สำหรับรูปภาพ

ทุกครั้งที่ใส่รูปภาพด้วยแท็ก <img> คุณ ต้อง ใส่แอตทริบิวต์ alt เสมอ

<!-- ดี: อธิบายว่ารูปคืออะไร เพื่อให้โปรแกรมอ่านให้คนตาบอดฟัง -->
<img src="dog.jpg" alt="สุนัขพันธุ์โกลเด้น รีทรีฟเวอร์กำลังคาบจานร่อน">

<!-- หากเป็นแค่รูปตกแต่งที่ไม่มีความหมาย ให้ใส่ alt ว่างๆ ไว้ -->
<img src="decorative-line.png" alt="">

Color Contrast (ความเปรียบต่างของสี)

สีตัวอักษรและสีพื้นหลังต้องมีความแตกต่างกันมากพอ เพื่อให้ผู้ที่มีสายตาเลือนราง หรือคนที่ใช้งานกลางแดดจ้าสามารถอ่านออกได้ มาตรฐาน WCAG (Web Content Accessibility Guidelines) กำหนดว่า Text ทั่วไปควรมีอัตราส่วน Contrast อย่างน้อย 4.5:1 คุณสามารถใช้เครื่องมือใน Chrome DevTools หรือเว็บตรวจ Contrast Checker ออนไลน์เพื่อทดสอบสีของคุณได้

Keyboard Navigation และ Focus Management

ผู้ใช้จำนวนมาก (รวมถึง Power Users) ท่องเว็บด้วยปุ่ม Tab บนคีย์บอร์ด เมื่อผู้ใช้กด Tab ไปที่ลิงก์หรือปุ่ม ต้องมีกรอบแสดงให้เห็นชัดเจนว่าโฟกัสอยู่ที่ไหน (Focus Ring)

/* อย่าทำแบบนี้เด็ดขาด! มันทำลาย a11y ของผู้ใช้คีย์บอร์ด */
button:focus, a:focus {
    outline: none; 
}

/* ถ้าจะเอา outline ออก ต้องใส่สไตล์อื่นทดแทนให้ชัดเจน */
button:focus-visible {
    outline: 3px solid #3498db;
    outline-offset: 2px;
}

หมายเหตุ: เราใช้ :focus-visible (ฟีเจอร์ใหม่) แทน :focus เพราะมันจะแสดงกรอบเฉพาะเวลาใช้คีย์บอร์ดกดเท่านั้น ถ้าคลิกด้วยเมาส์จะไม่ขึ้นกรอบรบกวนสายตา

ARIA Roles (WAI-ARIA)

บางครั้งเราต้องสร้าง UI ที่ซับซ้อนซึ่งไม่มี HTML tag รองรับโดยตรง (เช่น Custom Dropdown หรือ Modal) เราสามารถใช้ ARIA (Accessible Rich Internet Applications) attributes เพื่อเพิ่มข้อมูลเสริมให้ Screen reader เข้าใจได้

<!-- บอก Screen Reader ว่าแท็กนี้ทำหน้าที่เป็นปุ่ม แม้จะไม่ใช่ tag button -->
<div role="button" tabindex="0" aria-pressed="false">คลิกฉัน</div>

<!-- aria-hidden="true" ซ่อนจาก Screen Reader (แต่ตายังมองเห็น) -->
<i class="icon-search" aria-hidden="true"></i>
💡 Pro Tip: กฎข้อแรกของการใช้ ARIA คือ "อย่าใช้ ARIA ถ้ามี HTML Tag มาตรฐานให้ใช้อยู่แล้ว" (No ARIA is better than bad ARIA) การใช้ <button> ย่อมดีและมีบั๊กน้อยกว่าการพยายามเอา <div> มาแปะ ARIA ให้กลายเป็นปุ่ม!
