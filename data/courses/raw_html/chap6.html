ก้าวข้ามขีดจำกัดด้วย CSS Grid Layout

ในขณะที่ Flexbox เกิดมาเพื่อจัดการ Layout ใน 1 มิติ (แถว หรือ คอลัมน์ อย่างใดอย่างหนึ่ง) CSS Grid ถูกออกแบบมาสำหรับจัดการ Layout แบบ 2 มิติ (ทั้งแถวและคอลัมน์พร้อมกัน) Grid เป็นเครื่องมือจัดวาง Layout ที่ทรงพลังที่สุดใน CSS ณ ปัจจุบัน สามารถสร้างตาราง กริด หรือเลย์เอาต์หน้าเว็บที่ซับซ้อนได้อย่างเป็นระเบียบและโค้ดสะอาด

เริ่มต้นใช้งาน Grid

เช่นเดียวกับ Flexbox คุณต้องเปลี่ยนกล่องแม่ (Parent) ให้เป็น Grid Container ด้วย display: grid; จากนั้นคุณจะสามารถกำหนด "เส้นกริด" เพื่อสร้างเป็นตารางขึ้นมาได้

สร้างโครงสร้าง: Columns และ Rows

หัวใจหลักของการสร้าง Grid คือ Property สองตัวนี้:

  • grid-template-columns: กำหนดจำนวนและความกว้างของคอลัมน์
  • grid-template-rows: กำหนดจำนวนและความสูงของแถว
.grid-container {
    display: grid;
    grid-template-columns: 200px 1fr 1fr; /* มี 3 คอลัมน์: 200px, ส่วนที่เหลือหารสอง */
    grid-template-rows: 100px 300px; /* มี 2 แถว: 100px และ 300px */
    gap: 20px; /* ระยะห่างระหว่างช่องกริด ทั้งแถวและคอลัมน์ */
}

หน่วย fr (Fractional Unit) เป็นหน่วยพิเศษสำหรับ Grid หมายถึง "สัดส่วนพื้นที่ว่างที่เหลืออยู่" ในตัวอย่างด้านบน 1fr 1fr หมายความว่าพื้นที่ที่เหลือจาก 200px จะถูกแบ่งครึ่งเท่าๆ กัน

การจัดวาง Item ลงใน Grid (Grid Placement)

หลังจากสร้างโครงข่ายตารางแล้ว คุณสามารถสั่งให้ลูก (Grid Items) เข้าไปอยู่ในตำแหน่งที่ต้องการได้ โดยระบุเส้นเริ่มต้นและเส้นสิ้นสุด:

.header {
    grid-column: 1 / 4; /* ลากตั้งแต่เส้นคอลัมน์ที่ 1 ไปจนถึงเส้นที่ 4 (กินพื้นที่ 3 คอลัมน์) */
}
.sidebar {
    grid-row: 2 / 4; /* ลากตั้งแต่เส้นแถวที่ 2 ไปเส้นที่ 4 */
}

เทคนิคระดับสูง: Grid Template Areas

นี่คือฟีเจอร์ที่ทำให้ Grid พิเศษกว่าใคร คุณสามารถ "วาด" เลย์เอาต์ของคุณเป็นภาพด้วยข้อความ (ASCII Art) ผ่าน Property grid-template-areas ได้เลย:

.container {
    display: grid;
    grid-template-areas:
        "header header header"
        "sidebar main main"
        "footer footer footer";
}

/* แล้วกำหนดให้ลูกแต่ละตัวไปลงตามชื่อ area */
.header { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main { grid-area: main; }
.footer { grid-area: footer; }

วิธีนี้ทำให้โค้ดอ่านง่ายและปรับเปลี่ยน Layout สำหรับ Responsive ได้ง่ายสุดๆ เพียงแค่เปลี่ยนการเรียง "ชื่อ" ใน Media Query

💡 Flexbox หรือ Grid เลือกอะไรดี?
กฎง่ายๆ คือ: ใช้ Flexbox สำหรับส่วนประกอบเล็กๆ ภายในหน้า (UI Components) เช่น แถบเมนู, กลุ่มปุ่ม, หรือ Card ที่เรียงต่อกัน แต่ให้ใช้ Grid สำหรับการจัดโครงสร้างใหญ่ของหน้าเว็บ (Page Layout) เช่น การจัดวาง Header, Sidebar, Main Content, และ Footer
