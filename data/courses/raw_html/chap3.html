อะไรคือ The Box Model?

The Box Model เป็นหัวใจสำคัญที่สุดในการเข้าใจการจัดวาง Layout ใน CSS แนวคิดคือ ทุกๆ องค์ประกอบ (Element) บนหน้าเว็บจะเป็น "กล่องสี่เหลี่ยม" แม้ว่าคุณจะเห็นมันเป็นข้อความหรือวงกลมก็ตาม กล่องนี้ประกอบด้วย 4 ส่วนจากด้านในออกไปด้านนอก:

  1. Content: เนื้อหาหลักด้านในสุด (ข้อความ หรือ รูปภาพ) กำหนดขนาดได้ด้วย width และ height
  2. Padding: พื้นที่ว่างระหว่าง Content กับ Border เปรียบเสมือนกันกระแทกด้านในกล่อง
  3. Border: เส้นขอบของกล่อง ล้อมรอบ Padding และ Content
  4. Margin: พื้นที่ว่างด้านนอกสุด ผลักกล่องนี้ให้ออกห่างจากกล่องข้างเคียง (Element อื่นๆ)

ปัญหาคลาสสิกของ Box Model (The Box Model Problem)

ตามค่าเริ่มต้นของเบราว์เซอร์ เมื่อคุณกำหนด width: 100px; มันคือกำหนดให้ Content กว้าง 100px เท่านั้น ถ้าคุณเพิ่ม padding: 20px; และ border: 5px; เข้าไป ความกว้างรวมของกล่องจะกลายเป็น: 100 + 20(ซ้าย) + 20(ขวา) + 5(ซ้าย) + 5(ขวา) = 150px!

ปัญหานี้ทำให้การจัด Layout พังได้ง่ายมาก เพราะขนาดกล่องบวมเกินกว่าที่คุณกำหนดไว้ วิธีแก้ปัญหาที่ดีที่สุดและกลายเป็นมาตรฐานของโลกไปแล้วคือการใช้ box-sizing: border-box;

* {
    box-sizing: border-box;
    margin: 0;
    padding: 0;
}

เมื่อใช้ border-box กล่องที่กำหนด width: 100px; จะรวมเอา Padding และ Border ไว้ใน 100px นั้นเลย เนื้อหาด้านในจะถูกบีบให้เล็กลงแทน ทำให้กล่องไม่บวมเกินขนาดที่เราตั้งใจไว้

Display Property: พฤติกรรมของกล่อง

กล่องแต่ละประเภทมีพฤติกรรมการแสดงผลเริ่มต้นไม่เหมือนกัน ซึ่งควบคุมด้วย property display:

  • Block (display: block;): เช่น <div>, <h1>, <p> จะจองพื้นที่เต็มความกว้าง (100%) เสมอ และบังคับให้ Element ถัดไปขึ้นบรรทัดใหม่ กำหนด width และ height ได้
  • Inline (display: inline;): เช่น <span>, <a>, <strong> จะใช้พื้นที่กว้างเท่าที่จำเป็นตามเนื้อหาด้านใน ไม่สามารถ กำหนด width, height, หรือ margin-top/bottom ได้
  • Inline-block (display: inline-block;): ลูกครึ่งระหว่าง block และ inline เรียงต่อกันในบรรทัดเดียวได้ (เหมือน inline) แต่สามารถกำหนด width, height และ margin ได้ (เหมือน block) เช่น <button>
  • None (display: none;): ลบ Element นั้นออกจากหน้าเว็บไปเลย (กล่องหายไป และไม่กินพื้นที่ใดๆ)

Margin Collapse: พฤติกรรมแปลกๆ ที่ต้องรู้

เรื่องที่ทำให้มือใหม่ปวดหัวบ่อยๆ คือ Margin Collapse (การยุบรวมกันของ Margin) เมื่อคุณมี Element 2 ตัววางซ้อนกัน (บน-ล่าง) ตัวบนมี margin-bottom: 30px; และตัวล่างมี margin-top: 20px; ระยะห่างตรงกลางระหว่างสองกล่องนี้จะ ไม่ใช่ 50px แต่จะเป็นแค่ 30px (เบราว์เซอร์จะเลือกค่า Margin ที่ใหญ่ที่สุดมาใช้ และจับยุบรวมกัน) หมายเหตุ: เหตุการณ์นี้เกิดขึ้นเฉพาะ Top และ Bottom เท่านั้น ไม่เกิดกับ Left และ Right

💡 Pro Tip: เทคนิคการจัดกึ่งกลาง Element กลางหน้าจอแบบรวดเร็วที่สุดสำหรับ Block element ที่ระบุความกว้างแล้ว คือการใช้ margin: 0 auto; (Top/Bottom เป็น 0, Left/Right เป็น auto ซึ่งจะแบ่งพื้นที่ที่เหลือสองฝั่งให้เท่ากัน)
